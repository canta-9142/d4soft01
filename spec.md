# 要件定義

## 1. 概要

本アプリは、タスクをカードとしてキャンバス上に配置し、タスク同士を線で接続できるCanvas型ToDoアプリである。通常のToDoアプリのようにタスクの追加、編集、完了管理を行えるだけでなく、タスク間の関係を視覚的に表現できる点を特徴とする。

ユーザーは、レポート作成、授業課題、個人開発、イベント準備などの作業を複数のタスクに分解し、それらをキャンバス上で整理する。タスクカード同士を線でつなぐことで、関連作業、依存関係、作業の流れなどを表現できる。

本アプリにおける「深さ」は、固定された階層ではなく、ユーザーが選択した基準タスクから見た相対的な距離である。基準タスク自身を深さ0、基準タスクに直接つながるタスクを深さ1、さらにその先につながるタスクを深さ2として扱う。ユーザーは表示する最大深さを指定することで、基準タスク周辺の関連タスクだけを表示できる。

実装は生のHTML、CSS、JavaScriptで行い、ブラウザ上で動作するアプリとする。データ保存にはLocalStorageを使用し、ページを再読み込みしてもタスク、カード位置、接続関係、表示設定が復元されることを目指す。

## 2. 用語の定義
### 2.1. 描画内容に関するもの
- メニューバー: 画面上部に表示される、現在の状態や操作メニューなどを表示する領域。
- キャンバス: タスクカードを配置する領域。一つのアプリは複数のキャンバスを有することができる。
- タスク: 作業の単位を表すカード型オブジェクト。キャンバス上に配置される。一つのキャンバスは複数のタスクを有することができる。
- タスクカード: 描画要素としてのタスクをタスクそのものとは区別してタスクカードと呼称する場合がある。
- ステータス: タスクの状態を表す属性。未着手、進行中、完了済みの3種類がある。
- 接続線: タスクカード同士をつなぐ線。
- モード: ユーザーの操作状態を表す属性。通常モード、接続モード、編集モードがある。
- タスクのメニュー: タスクを右クリックすると表示されるメニュー。
- キャンバスのメニュー: キャンバスを右クリックすると表示されるメニュー。

### 2.2. 操作に関するもの
- キャンバス上の何もないところ: キャンバス上でタスクカードが配置されていない領域。タスクカードの外側の領域。
- タスクを登録する: タスクカードを新規作成し、キャンバス上に配置すること。通常モードにおいて行うことができる。
- タスクを編集する: 既存のタスクカードのタイトル、説明文、完了状態を変更すること。編集モードにおいて行うことができる。
- タスクを移動する: 既存のタスクカードの位置を変更すること。通常モードにおいて行うことができる。
- タスクを接続する: 既存のタスクカード同士を接続線でつなぐこと。接続モードにおいて行うことができる。
- タスクを削除する: 既存のタスクカードをキャンバス上から削除すること。削除時は、そのタスクにつながるすべての接続線が同時に削除される。通常モードにおいて行うことができる。
- 保存: 現在のキャンバスの状態をLocalStorageに保存すること。通常モードにおいて行うことができる。どれか一つでも保存条件を満たさないタスクが存在する場合は、保存に失敗する。
- 保存条件: 後述(〇.〇節)。
- 復元: LocalStorageに保存されたキャンバスの状態を読み込み、キャンバスを復元すること。キャンバスのメニューから、通常モードにおいて行うことができる。
- キャンバスを移動する: キャンバスの表示位置を変更すること。通常モードにおいて行うことができる。
- キャンバスを切り替える: 表示するキャンバスを変更すること。通常モードにおいて行うことができる。保存に失敗した場合、キャンバスの切り替えは実行されない。

## 3. 表示
### 3.1. 画面構成

- 上部: メニューバー
  - 左端にキャンバス切り替え用のハンバーガーメニュー
  - 現在の状態
  - タスクのステータスごとの個数
  - 右端に保存ボタン

- 全体: キャンバス
  - 方眼紙のような格子状の模様
  - そのキャンバスに属するタスクカード

### 3.2. キャンバス
- タスクカードを配置する領域

### 3.3. メニューバー
- ハンバーガーメニュー: キャンバスの切り替えや新規キャンバスの作成などの操作を提供する。
- 現在の状態: 現在のモード、検索条件、深さフィルターの状態(有効/無効、表示深さ)を表示する。
- タスクのステータスごとの個数: 合計のタスク数と、未着手・進行中・完了済みのそれぞれのタスク数を表示する。

### 3.4. タスクカード
#### 縮小
- タイトル、ステータスのみを表示。
- 拡大・編集モードよりも小さいサイズで表示する。

#### 拡大
- 選択中のタスクの表示形式。
- タイトル、ステータス、説明文を表示する。
- 大きさも少し拡大する。

#### 編集
- 編集中のタスクの表示形式。
- 拡大モードとほぼ同じ見た目。
- タイトル、ステータス、説明文を編集できる。

### 3.5. 接続線
- タスクカード同士をつなぐ線。
- タスクカードの中央と中央を結ぶ。
- 必ず向きを持ち、矢印でその向きを示す。
- 矢印の矢の部分は、線の中央に描画する。

## 4. 操作方法とその動作の定義
### 4.1. 操作の一覧

### 4.2. キーボード
#### 汎用
- Ctrl+A:       新規タスク登録
- Ctrl+Z:       操作の取り消し
- Ctrl+Y:       操作のやり直し
- Shift+↑/↓:    キャンバス切り替え
- Ctrl+S:       保存
- Ctrl+X:       接続モードの有効/無効切替
- Ctrl+F:       検索
- Ctrl+Shift+F: 検索条件クリア
#### 編集中
- Ctrl+C:       文字列コピー
- Ctrl+V:       文字列貼り付け
#### タスク選択中
- Ctrl+C:       選択タスクコピー
- Ctrl+V:       選択タスク貼り付け
- Ctrl+E:       選択タスク編集
- Ctrl+D:       選択タスク削除
- Delete:       選択タスク削除
- ↑/↓:          選択タスク切換え

### 4.3. マウス
- キャンバス上の何もないところをクリック: 新規タスク登録
- キャンバス上の何もないところをダブルクリック: 新規タスク登録
- キャンバス上の何もないところを右クリック: キャンバスのメニューを表示
- キャンバス上の何もないところをドラッグ: キャンバス移動
- タスクをクリック: タスクを選択
- タスクをダブルクリック: タスクを編集
- タスクを右クリック: タスクのメニューを表示
- タスクをドラッグ: タスクの移動
- タスクを右クリックでドラッグ: タスクの接続
- タスクにマウスホバー: タスクをホバーモード表示
- (キャンバスの切り替えは左上のハンバーガーメニューから行う)
